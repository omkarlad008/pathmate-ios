# UX & Accessibility

SwiftUI layouts are kept breathable and legible.

## HIG Choices
- Hit areas ≥ 44×44pt; generous vertical padding.
- Clear hierarchy with system typography.
- Subtle materials for contrast.

## Accessibility
- VoiceOver labels on primary actions (e.g., “Mark task done”).
- Headers marked where useful.
- Monospaced digits for progress.

## Interactions
- **Checklist**: swipe a row to **mark done/undone**.
- **Home**: toggle “Today’s plan” tiles.
- **Widget**: checkmark via ``MarkTaskDoneIntent``.
